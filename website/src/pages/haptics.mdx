# Haptics

Haptics are **on by default**. On supported devices, each toast triggers a small vibration pattern to make feedback feel more “native”. See also the [`<Toaster />` API](/toaster#haptics-api) for a concise prop list.

<Callout type="info">
  On desktop, vibration usually isn’t available—use **`hapticsDebug`** to hear the pattern as sound while developing.
</Callout>

## Quick usage

```tsx
import { Toaster } from 'web-haptics-toast';
import 'web-haptics-toast/dist/styles.css';

export function App() {
  return <Toaster />;
}
```

`haptics` defaults to `true`, so you only need to pass props when you want to change behavior.

## Props

### `haptics`

Enable or disable haptics for this toaster instance.

```tsx
<Toaster haptics={false} />
```

### `hapticsDebug`

When `true`, plays the pattern as **audio** on desktop (same idea as web-haptics debug). On phones that vibrate, you still get real haptics; debug can add sound on top depending on environment.

```tsx
<Toaster haptics hapticsDebug />
```

### `hapticsShowSwitch`

Shows the small **web-haptics** on-screen toggle (same as `showSwitch` in web-haptics) so people can turn feedback off without wiring your own UI.

```tsx
<Toaster haptics hapticsShowSwitch />
```

### `hapticPatternMap`

Map each **toast type** to a **named pattern**. Pattern names must exist in the bundled presets (same as [web-haptics-main](https://github.com/lochie/web-haptics)): `success`, `error`, `warning`, `light`, `medium`, `heavy`, `soft`, `rigid`, `selection`, `nudge`, `buzz`.

Defaults (when you don’t pass a map) include e.g. `success` → `success`, `error` → `error`, `warning` → `warning`, `info` / `loading` / `default` → `light`.

```tsx
<Toaster
  haptics
  hapticPatternMap={{
    success: 'success',
    error: 'error',
    warning: 'warning',
    info: 'selection',
    default: 'light',
  }}
/>
```

## Programmatic helpers

You can also use the same primitives outside the toaster:

- **`isHapticsSupported`** – whether `navigator.vibrate` is available.
- **`triggerHaptic(name, options?)`** – fire a preset by name; use `{ debug: true }` to hear it on desktop.
- **`WebHaptics`** / **`defaultPatterns`** – advanced use; same model as web-haptics-main.

## Tips

- Keep patterns subtle for frequent toasts (e.g. `selection` / `light`).
- Use stronger patterns for rare, important outcomes (`success` / `error`).
- Offer a user-facing way to disable haptics where your product has accessibility or preference settings.
