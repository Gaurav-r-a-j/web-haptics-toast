# Migrating from Sonner

**web-haptics-toast** is a **standalone** package: you install it once and get Sonner-compatible toasts **plus** a built-in haptics engine—no second dependency for vibration or presets.

It tracks [Sonner](https://sonner.emilkowal.ski/)’s public API (``toast`()`, `<Toaster />`, styling hooks). On top of that you get **haptics** (on by default) and an optional **`system`** layer for dedupe and dismiss reasons ([Getting Started](/getting-started), [Toaster](/toaster)).

Pick one path below.

## Option A — Zero code changes (keep `from 'sonner'`)

Install this package **under the name `sonner`**. Your imports and re-exports stay identical.

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']}>
  <Tabs.Tab>
    ```bash
    npm uninstall sonner && npm install sonner@npm:web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    pnpm remove sonner && pnpm add sonner@npm:web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    yarn remove sonner && yarn add sonner@npm:web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    bun remove sonner && bun add sonner@npm:web-haptics-toast
    ```
  </Tabs.Tab>
</Tabs>

- Keep: `import { Toaster, toast } from 'sonner'`
- Keep: `import 'sonner/dist/styles.css'` — the aliased package still exposes the same stylesheet path as Sonner.

<Callout type="info">
  **Haptics** run automatically on supported devices. To match old Sonner behavior (no vibration), use `<Toaster haptics={false} />` once at your app root.
</Callout>

## Option B — Use the `web-haptics-toast` package name

Useful when you want the dependency name to match the project and to avoid npm aliases.

<Steps>

### Remove Sonner, add web-haptics-toast

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']}>
  <Tabs.Tab>
    ```bash
    npm uninstall sonner && npm install web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    pnpm remove sonner && pnpm add web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    yarn remove sonner && yarn add web-haptics-toast
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    bun remove sonner && bun add web-haptics-toast
    ```
  </Tabs.Tab>
</Tabs>

### Update imports

Replace every Sonner import:

```diff
- import { Toaster, toast } from 'sonner';
- import 'sonner/dist/styles.css';
+ import { Toaster, toast } from 'web-haptics-toast';
+ import 'web-haptics-toast/dist/styles.css';
```

Search your repo for `from 'sonner'`, `from "sonner"`, and `sonner/dist/styles.css`.

### Optional: barrel files and tests

If you re-export from a shared `notifications.ts` (or similar), point that file at `web-haptics-toast` so the rest of the app keeps the same internal imports.

</Steps>

## After you migrate

| Topic | Notes |
| :---- | :---- |
| **Single package** | Components, CSS, and haptics (`WebHaptics`, presets) are **all** in `web-haptics-toast`—no extra haptics dependency. |
| **Haptics** | Default `true` on `<Toaster />`. Plain ``toast`('message')` uses the **default** pattern when haptics are on; pass `{ haptics: false }` on a call to skip. [Haptics](/haptics) — `hapticsDebug`, `hapticPatternMap`, `hapticsShowSwitch`, `triggerHaptic`. |
| **`system` prop** | Opt-in dedupe (`dedupeKey`) and `onClose` with a dismissal **reason**. Off by default so Sonner-like behavior is unchanged. [Getting Started](/getting-started), [Toaster](/toaster#system-production-behaviors). |
| **Types** | Published with the library; Sonner-shaped `ToasterProps` and `toast` options, plus haptics-related props. |
| **Styling** | Same `toastOptions`, `classNames`, `unstyled` patterns as Sonner. [Styling](/styling). |

## Checklist

- [ ] Dependencies updated (alias **or** renamed package).
- [ ] JS/TS imports and **CSS** import path updated if you chose Option B.
- [ ] `<Toaster />` still mounted once near the root.
- [ ] Decide on **haptics** (`true` default vs `haptics={false}`).
- [ ] If you need **dedupe** or **onClose(reason)**, enable `<Toaster system={{ … }} />` as in [Getting Started](/getting-started).

If something behaves differently from Sonner after migration, open an issue on the [GitHub repo](https://github.com/Gaurav-r-a-j/web-haptics-toast) with a minimal repro.
